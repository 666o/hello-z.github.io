这个博客的出生过程

发表于 2019-11-05 更新于 2019-12-25 阅读次数：

Anyway,一切从简.

这个博客拿来做笔记、积累错误解决经验

本来是用的Nginx+Django来搭建的个人博客，但是服务器到期外加不想再折腾再加一眼相中Hexo+Next的极简风格，决定重头做个简单博客

言归正传，gogogo

环境

如果有了以上的部分，便开始下一步。

本地Hexo初始化

创建博客目录

选好地址，创文件夹，例如在D盘创建 mysite 的文件夹

Hexo初始化

进入创建好的文件夹
右键打开 Git Bash
输入 $ hexo init

其实现在就搭建完毕了

生成并运行博客

输入 $ hexo g 生成静态页面

然后
$ hexo s
运行

打开浏览器，在地址输入 http://localhost:4000/

我们就可以看到激动人心的界面啦！

购买皮肤，充值信仰

虽然创建好了博客，是不是觉得它不是你想象中的那个她，没事，让我们来购买皮肤

选择主题

本站的主题是 Next-Muse

Github 链接：hexo-theme-next https://github.com/theme-next/hexo-theme-next

当然你可以选择其他的主题，操作和以下操作一样

下载主题

在网站根目录右键打开 Git Bash，输入以下命令将主题下载到本地

$ git clone https://github.com/theme-next/hexo-theme-next themes/next

启用主题

先更改根目录下 _config.yml 文件

找到 Extensions

改为

换装完毕

在 Git Bash 中输入
hexo clean && hexo g && hexo s

然后打开浏览器，在地址输入 http://localhost:4000/

换装完毕,还不是美滋滋

部署到GitHub

创建仓库

首先新建一个仓库，项目名称为 用户名.github.io ，其余默认

添加公钥

先在本地创建公钥

$ ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

记住生成的文件夹位置哦

然后在项目的设置中添加公钥

最后配置库名

打开根目录下_config.yml，找到deploy

修改为

最后提交

$ hexo clean && hexo g && hexo d

大功告成

正常情况下访问 yourname.github.io就可以看到你的博客啦

总结

到这里，我们整个的建立和部署都已经基本完成了

如果你有什么不明白的地方，或者遇到了什么问题

那就要恭喜你了

因为本站与百度达成了战略合作关系

你遇到的所有问题都可以直接在百度上免费搜索！是免费的哦！
