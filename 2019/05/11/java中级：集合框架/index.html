java中级：集合框架

发表于 2019-05-11 更新于 2020-10-03 分类于 java 阅读次数：

如题

ArrayList

数组的局限

数组有数组的局限性

例如：长度是固定的，超过了就会越界
声明的数量用不了就浪费了

用ArrayList存放多个对象

为了解决数组的局限性，引入容器类的概念。 最常见的容器类就是
ArrayList

容器的容量“capacity”会随着对象的增加，自动增长
只需要不断往容器里增加英雄即可，不用担心会出现数组的边界问题。

容器的常用方法

1.增加
add 有两种用法
第一种是直接add对象，把对象加在最后面
heros.add(new Hero("hero " + i));

第二种是在指定位置加对象
heros.add(3, specialHero);

2.判断对象是否在容器中

contains
heros.contains(target)

3.获取指定位置
get(index)

如果输入的下标越界，一样会报错

4.获取对象的下标
indexOf（target)

5.删除
remove
remove可以根据下标删除ArrayList的元素
remove(index)
也可以根据对象删除
remove(hero)

6.替换
set
set(index)

7.获取容器大小
size
size(ArrayList)

8.转换为数组
toArray
toArray(ArrayList)

9.把另一个容器所有对象都加进来
addALL
ArrayList.addALL(newArrayList)

10.清空
clear
ArrayList.clear()

List接口

ArrayList实现了接口List
常见的写法会把引用声明为接口List类型
注意：是java.util.List,而不是java.awt.List

泛型Generic

不指定泛型的容器，可以存放任何类型的元素
指定了泛型的容器，只能存放指定类型的元素以及其子类

List genericheros = new ArrayList();

遍历容器

1.for
利用下标遍历

2.迭代器

3.增强版for

其他集合

LinkedList

序列分先进先出FIFO,先进后出FILO
FIFO在Java中又叫Queue 队列
FILO在Java中又叫Stack 栈

与ArrayList一样，LinkedList也实现了List接口

意味着诸如add,remove,contains等方法一样使用

除了实现了List接口外，LinkedList还实现了双向链表结构Deque
LinkedList.addLast(Object)//在尾部插入对象
LinkedList.addFirst(Object)//在头部插入对象
LinkedList.getLast()//获取尾部对象
LinkedList.removeFirst()//取出头部对象

LinkedList 除了实现了List和Deque外，还实现了Queue接口(队列)。
Queue是先进先出队列 FIFO，常用方法：
offer 在最后添加元素
poll 取出第一个元素
peek 查看第一个元素

ArrayList与LinkedList的区别

ArrayList 插入，删除数据慢
LinkedList， 插入，删除数据快
ArrayList是顺序结构，所以定位很快，指哪找哪。 就像电影院位置一样，有了电影票，一下就找到位置了。
LinkedList 是链表结构，就像手里的一串佛珠，要找出第99个佛珠，必须得一个一个的数过去，所以定位慢

HashMap

HashMap储存数据的方式是—— 键值对

对于HashMap而言，key是唯一的，不可以重复的。
所以，以相同的key 把不同的value插入到 Map中会导致旧元素被覆盖，只留下最后插入的元素。
不过，同一个对象可以作为值插入到map中，只要对应的key不一样

HashSet

Set中的元素，不能重复

Set中的元素，没有顺序。
严格的说，是没有按照元素的插入顺序排列
换句话说，同样是插入0-9到HashSet中， 在JVM的不同版本中，看到的顺序都是不一样的。
所以在开发的时候，不能依赖于某种臆测的顺序，这个顺序本身是不稳定的

Set不提供get()来获取指定位置的元素

所以遍历只能使用迭代器或者增强型for循环

通过观察HashSet的源代码
可以发现HashSet自身并没有独立的实现，而是在里面封装了一个Map.
HashSet是作为Map的key而存在的
而value是一个命名为PRESENT的static的Object对象，因为是一个类属性，所以只会有一个。

Collection

Collection是一个接口

Collections是一个类，容器的工具类,就如同Arrays是数组的工具类

1.翻转
reverse
Collections.reverse(List);

2.混淆
shuffle
Collections.shuffle(List);

3.排序
sort
Collections.sort(List);

4.交换
swap
Collections.swap(List,index1,index2);

5.滚动
Collections.rotate(List,lenth);

6.线程安全化
List synchronizedNumbers = (List) Collections.synchronizedList(numbers);

HashMap和Hashtable的区别

HashMap和Hashtable都实现了Map接口，都是键值对保存数据的方式
区别1：
HashMap可以存放 null
Hashtable不能存放null
区别2：
HashMap不是线程安全的类
Hashtable是线程安全的类
