Mybatis入门：基础配置

发表于 2019-06-10 更新于 2019-12-29 阅读次数：

一些MyBatis的基础配置介绍

概述

MyBatis配置的所有元素如下所示：

注意：MyBatis的配置项不能颠倒，不能颠倒，不能颠倒！

properties 属性

properties属性可以给系统配置一些运行参数。
MyBatis提供了三种方式使用properties：

  • property子元素
  • properties文件
  • 程序代码传递

property 子元素

举个栗子

在properties中配置好了之后在中引入这个参数

如果类似的属性参数有成千上百，可以选择properties文件

properties文件

首先properties文件的格式就是 键值对

例如
jdbc.properties：

然后再：
<properties resource="jdbc.properties"/>

就完事了

程序代码传递

在真实的生产环境中，数据库的用户密码是对开发人员保密的。运维需要把密码加密成为密文后，配置到properties文件中。
就可以先使用解密的函数再将解密后的字符充值到properties属性中。

最后，三种方式是有优先级的，会根据优先级来覆盖原先配置的属性值

最优先的是程序代码传递，然后是文件，最后是子元素

settings 设置

settings是MyBatis中最复杂的配置，他能深刻影响MyBatis底层的运行，但是在大部分情况下使用默认值便可运行，所以在大部分情况下不需要大量配置它，只需要修改一些常用的规则即可，比如自动映射，驼峰命名映射、级联规则、是否启动缓存、执行器类型等

配置项 作用 配置选项说明 默认值
cacheEnabled 该配置影响所有映射器中配置缓存的全局开关 true\false true
lazyLoadingEnabled 延迟记载的全局开关。 true\false false
defaultStatementTimeout 设置超时时间，它决定驱动等待数据库响应的时间 任何正整数 Not Set(null)
multipleResultSetEnabled 是否允许单一语句返回多结果集 true\false true
useGeneratedKeys 允许JDBC支持自动生成主键，需要驱动兼容。如果设置为true，则这个设置强制使用自动生成主键 true\false false
autoMappingBehavior 指定MyBatis应如何自动映射到字段或者属性。NONE表示取消，PARTIAL表示只会自动映射，没有定义嵌套结果集和映射结果集，FULL会自动映射任意复杂的结果集 NONE,PARTIAL,FULL PARTIAL
defaultFetchSize 设置数据库驱动程序默认返回的条数限制，此参数可以重新设置 任何正整数 Not Set(null)
mapUnderscoreToCamelCase 是否开启自动驼峰映射规则映射 true\false false

配置项很多，我这里也仅仅列了一点出来，但是真正用到的不会太多。

typeAliases 别名

由于类的全限定名称很长，需要大量使用的时候，总写那么长的名称不方便。
在MyBatis中允许定义一个简写来代表这个类，这就是别名

注意，MyBatis中别名不区分大小写

系统定义别名

在MyBatis的初始化过程中，系统自动初始化了一些别名

如：

别名 Java类型 是否支持数组
_byte byte 是
_long long 是
_其他基础数据类型 其他基础数据类型 是
string String 是
其他包装类型 其他包装类型 是
date Date 是
map Map 否
hashmap HashMap 否
list List 否
arraylist ArrayList 否
collection Collection 否
iterator Iterator 否
ResultSet ResultSet 否

自定义别名

在程序中有的对象需要大量的重复使用，这个时候可以使用自定义别名的方式

typeHandler 类型转换器

一般而言，Java类型和数据库类型之间的转换是不用操心的，mybatis会自动进行处理，只有一些自定义的枚举类需要我们去自定义typeHandler.

environments 运行环境

主要的作用是配置数据库信息，它可以配置多个数据库，一般而言只需要配置一个。
它下面又分为两个可配置的元素：事务管理器、数据源。
在实际的工作中，大部分会使用Spring对数据源和数据库的事务进行管理
