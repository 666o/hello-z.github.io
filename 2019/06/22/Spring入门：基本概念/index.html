Spring入门：基本概念

发表于 2019-06-22 更新于 2020-10-03 分类于 java 阅读次数：

Java世界中最为成功的框架

开始学习Spring框架，它最为核心的观念是IoC（控制反转）和AOP（面向切面编程）。

其中IOC是Spring的基础，AOP是其重要的功能

Spring Ioc

控制反转：通过一种描述并通过第三方去产生或获取特定对象的方式

我的理解是，等同于自己放弃了创建对象的能力而转身向Spring索要，Spring承担了根据描述创建对象的责任

在Spring中，我们可以在Spring IOC容器中获取到各种发布在其中的Bean，并且通过描述可以得到它

如何获取Bean

先创建一个配置bean的xml文件（注解配置其实更方便）：

sprint_configbean.xml

这样就定义了一个Bean，然后再初始化IOC容器，再通过IOC容器获取对象

完事

Spring AOP

先贴一段别人的理解

面向切面编程（AOP是Aspect Oriented Program的首字母缩写），我们知道，面向对象的特点是继承、多态和封装。而封装就要求将功能分散到不同的对象中去，这在软件设计中往往称为职责分配。实际上也就是说，让不同的类设计不同的方法。这样代码就分散到一个个的类中去了。这样做的好处是降低了代码的复杂程度，使类可重用。
但是人们也发现，在分散代码的同时，也增加了代码的重复性。什么意思呢？比如说，我们在两个类中，可能都需要在每个方法中做日志。按面向对象的设计方法，我们就必须在两个类的方法中都加入日志的内容。也许他们是完全相同的，但就是因为面向对象的设计让类与类之间无法联系，而不能将这些重复的代码统一起来。
也许有人会说，那好办啊，我们可以将这段代码写在一个独立的类独立的方法里，然后再在这两个类中调用。但是，这样一来，这两个类跟我们上面提到的独立的类就有耦合了，它的改变会影响这两个类。那么，有没有什么办法，能让我们在需要的时候，随意地加入代码呢？这种在运行时，动态地将代码切入到类的指定方法、指定位置上的编程思想就是面向切面的编程。
一般而言，我们管切入到指定类指定方法的代码片段称为切面，而切入到哪些类、哪些方法则叫切入点。有了AOP，我们就可以把几个类共有的代码，抽取到一个切片中，等到需要时再切入对象中去，从而改变其原有的行为。这样看来，AOP其实只是OOP的补充而已。OOP从横向上区分出一个个的类来，而AOP则从纵向上向对象中加入特定的代码。有了AOP，OOP变得立体了。如果加上时间维度，AOP使OOP由原来的二维变为三维了，由平面变成立体了。从技术上来说，AOP基本上是通过代理机制实现的。
AOP在编程历史上可以说是里程碑式的，对OOP编程是一种十分有益的补充.

重点在于：在运行时，动态地将代码切入到类的指定方法、指定位置上的编程思想就是面向切面的编程

一些术语

1.切面（Aspect）：切面就是怎样的环境中工作，它可以定义后面的各类通知，切点和引入等内容，然后SpringAOP会将其定义的内容织入到约定的流程中

2.通知（Advice）：通知是切面开启后，切面里的方法。它根据再代理对象真实方法调用前，后的顺序与逻辑划分

3.切点（Pointcut）：这是一个告诉SpringAOP在什么时候启动拦截并织入对应的流程中，并不是所有的开发都需要AOP，它往往通过正则进行限定

4.连接点 (join point):连接点对应的是具体需要拦截的东西，比如通过切点的正则去判断哪些方法是连接点，从而织入对应的通知

简单使用

这里使用@AspectJ注解开发

1.选择连接点
Spring是方法级别的AOP，所以我们也主要是以某个类的某个方法作为连接点。

这里我们选择printRole方法

2.创建切面

对于动态代理的概念而言，它就如同一个拦截器，在Spring中只要使用了@Aspect注解一个类，那么SpringIOC容器就会认为它是一个切面了

注解 通知 备注
@Before 在被代理对象的方法之前调用 前置通知
@After 在被代理对象的方法之后调用 后置通知
@AfterReturning 在被代理对象的方法正常返回后调用 返回通知，要求方法执行过程中没有发生异常
@AfterThrowing 在被代理对象的方法抛出异常后调用 异常通知，要求被代理对象的方法执行过程中出异常
@Aroud 将被代理对象的方法封装起来，并用环绕通知取代它 环绕通知，它将覆盖原有方法
