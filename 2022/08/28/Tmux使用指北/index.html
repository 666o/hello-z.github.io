Tmux使用指北

这里引用 Tmux 在 GitHub 上的简介来介绍它自己

tmux is a terminal multiplexer. It lets you switch easily between several programs in one terminal, detach them (they keep running in the background) and reattach them to a different terminal.

简单来说，Tmux 能帮助你构建多个长期的终端，而不是随着你的ssh链接的关闭而关闭，这将非常的实用，当你需要在服务器上部署大量项目时。

官方文档及优秀教程：

  • 官方wiki：https://github.com/tmux/tmux/wiki
  • 阮一峰的网络日志：https://www.ruanyifeng.com/blog/2019/10/tmux.html
  • 无涯教程：http://www.learnfk.com/linux/linux-command-tmux.html

安装

参见官方安装教程：tmux installing

概念

在 Tmux 的逻辑中，我理解为四个层级 Server => Session => Window => Pane

  • Server：Tmux 服务进程，包含所有 Session 信息

  • Session：当我们用ssh连接服务器时，这种用户与计算机临时的交互叫做Session（会话），会话的重要特点是，会话中启动的进程与会话是联系在一起的。Tmux 中的一个Sessioin就像是一次ssh连接，不同的是他能保存和脱离会话，将Session放在后台持续运行。

  • Window：相当于一个工作区，包含很多 Pane ,可以针对每种组件分一个Window。

  • Pane：是在 Window 中的小分屏。

我个人习惯一个项目开一个 Session ，然后项目中的每个组件开一个 Window ，根据组件的情况来开 Pane ,例如日志的监控一个 Pane, 进程的启动一个 Pane

外部常用命令

ps:这里的外部是指在外部终端窗口，与 Tmux 内部 Session 做区分

启动新Session

恢复已有session

tmux a -t session_name

or

tmux at -t session_name

查询现有session

tmux ls

关闭已有session

tmux kill-session -t session_name

关闭tmux Server

tmux kill-server

ps：有时候更改 Tmux 配置不生效，就要使用该命令进行重启

重命名会话

tmux rename-session -t old_session_name new_session_name

内部常用命令

所谓内部命令，就是进入 Tmux 中，并按下前缀按键后的命令，一般前缀键为Ctrl+b. 可在配置文件中更改前缀按键

在这里前置命令使用：prefix 来代替

session

window

pane

配置文件

Tmux 是开箱即用的，不需要特别配置，以下只是为了方便个人使用

配置文件的位置放在 ~/.tmux.conf ，避免影响他人，修改之后需要在session中重新加载 or 重启 tmux服务

插件

如同配置文件，Tmux 自带功能已经很好用了，没有额外的需要就没必要装插件

Tmux-Resurrect

Tmux 虽然好用，但是也有一个缺点，就是关机之后打开的session就全消失了

虽说服务器一般不会关机，但是根据笔者的经验，什么都说不准

这个时候你就需要Tmux-Resurrect插件来了，地址如下

tmux-logging

如名，记录tmux中所有操作、历史记录
