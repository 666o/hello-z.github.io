2.0 用户操作

2.1 初始化

  • git init：为当前目录生成.git文件
  • git init filepath：为文件路径生成文件且生成.git文件
  • git init --bare filepath：为文件路径生成文件且生成裸仓库
  • git clone <repo> <directory>：从现有Git仓库中拷贝项目

关于 裸仓库：

  • 什么是GIT裸库

  • 普通库与裸库的区别

  • 该如何使用一个裸库

当需要协作时，在中央服务器上使用裸仓库，管理起来会方便很多

2.2 status

查看当前状态：git status

status命令用于显示工作目录和暂存区的状态。使用此命令能看到那些修改被暂存到了, 哪些没有, 哪些文件没有被Git tracked到。git status不显示已经commit到项目历史中去的信息。看项目历史的信息要使用git log.

可以使用 -s 参数来获得简短的输出结果

2.3 add

git add pathspec

该命令在工作树中找到的当前的内容以更新索引，为下一次提交准备暂存的内容。它通常会整体添加现有路径的当前内容，但是通过某些选项，它也可以用于仅添加对工作树文件所做的部分更改，或删除工作树中不存在的路径。

“索引”保存着工作树内容的快照，该快照作为下一次提交的内容。因此，在对工作树进行任何更改之后，以及在运行commit命令之前，必须使用add命令将所有新文件或修改过的文件添加到索引中。

提交前可以多次执行此命令。它仅在运行add命令时添加指定文件的内容；如果要在下一次提交中包含后续更改，则必须再次运行git add将新内容添加到索引中。

2.4 commit

git commit -m "message"

创建一个新的提交，其中包含索引的当前内容和描述更改的给定日志消息。新提交是 HEAD 的直接子节点，通常是当前分支的尖端，并且分支被更新以指向它。

2.5 reset

git-reset - 重置当前HEAD到指定的状态

  • git reset --hard [HEAD] 将工作区和暂存区的文件回退到指定版本，丢弃工作区和暂存区的所有改动。
  • git reset --soft [HEAD] 将现在到指定版本的改动回退到暂存区，常见为 HEAD^ 意为回退到上个版本，其中改动转到暂存区，主要用于修改上次提交。不会将丢弃现有工作区与暂存区的改动。
  • git reset --mixed [HEAD] 将现在到指定版本的改动回退到工作区，比soft更彻底。会将暂存区清空，工作区改动会保留。

不同模式的最大区别是在于工作区与暂存区文件的存留，总的来说，reset更像是goto，前往某个版本。

个人功能分支使用 reset 直接进行回退而不留下记录

main分支建议使用 revert 进行回退，留下记录

单个文件的回退可以用 checkout – file 将工作区的改动丢弃

2.6 branch

几乎每一种版本控制系统都以某种形式支持分支，一个分支代表一条独立的开发线。

使用分支意味着你可以从开发主线上分离开来，然后在不影响主线的同时继续工作。

  • git branch 查看本地分支
  • git branch -c branch_name 新建分支
  • git branch -d branch_name 删除指定分支

2.7 checkout

git-checkout - 切换分支或恢复工作树文件,很百搭的一个命令，功能太多以至于现在在被拆分

  • 切换分支：git checkout branch_name
  • 创建新分支并切换：git checkout -b branch_name
  • 放弃工作区中的修改：git checkout -- filename

2.8 merge

git-merge - 将两个或多个开发历史连接在一起

git merge branch_name

3 远程基本操作

3.1 remote

  • 显示所有远程仓库：git remote -v
  • 显示某个版本库信息：git remote show [remote]
  • 删除远程库：git remote rm name
  • 添加远程仓库：git remote add [name] [url]

3.2 fetch

从远程获取代码库

git fetch

3.3 pull

git pull 命令用于从远程获取代码并合并本地的版本。

git pull 其实就是 git fetch 和 git merge FETCH_HEAD 的简写

git pull <远程主机名> <远程分支名>:<本地分支名>

执行过一次之后可以只写 git pull

3.4 push

git push 命令用于从将本地的分支版本上传到远程并合并。

git push <远程主机名> <本地分支名>:<远程分支名>

如果本地分支名与远程分支名相同，则可以省略冒号：

git push <远程主机名> <本地分支名>
