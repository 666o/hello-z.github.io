注意：Dockerfile 的指令每执行一次都会在 docker 上新建一层。所以过多无意义的层，会造成镜像膨胀过大。

1.2 ENV

设置环境变量指令

第一种用法用于设置单个变量(第一个空格前为key，之后都是value,包括后面的空格)，第二种用于同时设置多个变量(空格为分隔符，value中包含空格时可以用双引号把value括起来，或者在空格前加反斜线)，当需要同时设置多个环境变量时推荐使用第二种格式。这些环境变量可以通过docker run命令的–env参数来进行修改。

当环境变量很多时， 可以使用env_file指定对应的变量文件

1.5 ARG

在Dockerfile中使用，仅仅在build docker image的过程中（包括CMD和ENTRYPOINT）有效，在image被创建和container启动之后，无效。

如果在Dockerfile中使用了ARG但并未给定初始值，则在运行docker build的时候未指定该ARG变量，则会失败。在FROM 之后RUN 前要重新定义前面FROM前的ARG

1.6 WORKDIR

WORKDIR指令用来设置Dockerfile中任何使用目录的命令的当前工作目录，此目录如果不存在就会被自动创建，即使这个目录不被使用

指明通过docker exec -it 进入容器内部后，默认进入的工作目录

WORKDIR /mydir

1.7 VOLUME

容器运行时应该尽量保持容器存储层不发生写操作，对于数据库类需要保存动态数据的应用，其数据库文件应该保存于卷(volume)中。为了防止运行时用户忘记将动态文件所保存目录挂载为卷，在Dockerfile 中，我们可以事先指定某些目录挂载为匿名卷，这样在运行时如果用户不指定挂载，其应用也可以正常运行，不会向容器存储层写入大量数据。

volume只是指定了一个目录，用以在用户忘记启动时指定-v参数也可以保证容器的正常运行。比如mysql，你不能说用户启动时没有指定-v，然后删了容器，就把mysql的数据文件都删了，那样生产上是会出大事故的，所以mysql的dockerfile里面就需要配置volume，这样即使用户没有指定-v，容器被删后也不会导致数据文件都不在了。还是可以恢复的。

如果-v和volume指定了不同的位置,会以-v设定的目录为准，其实volume指令的设定的目的就是为了避免用户忘记指定-v的时候导致的数据丢失，那么如果用户指定了-v，自然而然就不需要volume指定的位置了。

1.8 COPY & ADD

COPY & ADD指令能够将构建命令所在的主机本地的文件或目录，复制到镜像文件系统。

add除了复制，还具备添加远程文件功能（文件以http://开头），类似于linux的wget

add是增强版的copy，add自带解压，copy不带解压。

ADD指令不支持认证，从远程获取资源如果需要认证（交互），则只能使用RUN wget或RUN curl替代。

如果不是需要解压，优先使用COPY命令，效率更高更节省资源。

1.9 EXPOSE

指令用于标明，这个镜像中的应用将会侦听某个端口，并且希望能将这个端口映射到主机的网络界面上。但是，为了安全，docker run命令如果没有带上响应的端口映射参数，docker并不会将端口映射到宿主机。

只是一个声明 规范化Dockerfile,使对外暴露的端口一目了然

1.10 CMD & ENTRYPOINT

都是执行命令，但是它们的执行时机是不同的

  • RUN 仅在Build构建时执行的命令
  • CMD 用来设置容器启动后默认执行的命令(参数)，它可被docker run后面的命令参数替换
  • ENTRYPOINT 容器run启动时执行的命令
  1. CMD命令，在Dockerfile中，可以单独使用，但不一定会别执行，也可以作为enterpoint命令的一部分，作为参数使用。
  2. Dockerfile中如果存在多个CMD指令，仅最后一个生效。
  3. ENTRYPOINT如果在Dockerfile中有多个，则仅最后一个会被运行。
  4. 如果在Dockerfile中即使用了ENTRYPOINT，也使用了CMD，两者会被合并执行，CMD作为前者的默认参数使用
  5. docker run命令末尾，可以添加参数覆盖Dockerfile中CMD的默认参数，更为灵活。例： docker run nginx:run ls 最后的 ls优先级比较高，会覆盖CMD，相当于把它替换为CMD [“-aux”]

2. 镜像操作

当运行容器时，使用的镜像如果在本地中不存在，docker 就会自动从 docker 镜像仓库中下载，默认是从 Docker Hub 公共镜像源下载。

3.1 构建镜像

docker build -t 镜像名：版本号 上下文位置（默认为 .）

会默认使用当前目录的Dockerfile进行编译镜像

[OPTIONS]:

  • -t: 指定镜像的名字

  • -f: 显示指定Dockerfile

  • --build-arg=[]: 设置镜像创建时的变量

  • -m: 设置内存最大值

  • --quiet, -q: 安静模式，成功后只输出镜像 ID

  • --no-cache: 创建镜像的过程不使用缓存

  • --network: 默认 default。在构建期间设置RUN指令的网络模式

3.2 查看镜像列表

docker images or docker image ls

3.3 获取新镜像

docker pull imagename:tag

3.4 查询镜像

  • Docker Hub 搜索镜像

  • docker search keywords 搜索镜像

3.5 删除镜像

docker rmi hello-world

3. 容器操作

3.1 启动容器

docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

参数：

  • -e: 设置环境变量

  • --name: 为容器指定一个名称

  • -t: 为容器重新分配一个伪输入终端，通常与 -i 同时使用

  • -i: 以交互模式运行容器，通常与 -t 同时使用

  • -d: 后台运行容器，并返回容器ID

  • -p: 指定端口映射，格式为：主机(宿主)端口:容器端口 -p 80:8080

  • -v: 宿主机目录:容器目录。将宿主机目录挂载到容器内

  • --net=default: 指定容器的网络连接类型，支持 bridge/host/none/container: 四种类型

  • --restart

    • unless-stopped 在容器退出时总是重启容器，但是不考虑Docker守护进程启动时就已停止的
    • no(不自动重启)
    • on-failure:3 在容器非正常退出时（退出状态非0），才会重启容器，最多重启三次
    • always 在容器退出时总是重启容器

  • --health-cmd: 运行命令以检查健康

启动已停止的容器：docker start 容器ID

3.2 停止容器

docker stop CONTAINER

3.3 删除容器

docker rm CONTAINER

3.4 查案容器日志

[OPTIONS]：

  • -f: 跟踪日志输出
  • --since: 显示某个开始时间的所有日志
  • t: 显示时间戳
  • --tail：仅列出最新N条容器日志

3.5 进入&退出容器

进入：

  • docker exec：docker exec -it CONTAINER /bin/bash
  • docker attach： docker attach CONTAINER

退出：

  • exec 进入的：cmd: exit
  • attach进入的：ctrl+p+q

3.6 查看所有容器

docker ps [OPTIONS]

[OPTIONS]：

  • -a: 显示所有的容器，包括未运行的
  • -f: 根据条件过滤显示的内容
  • -l: 显示最近创建的容器
  • -n: 列出最近创建的n个容器
  • -q: 静默模式，只显示容器编号
  • -s: 显示总的文件大小

3.7 查看容器

查看内部进程：docker top CONTAINER

查看底层信息：docker inspect CONTAINER

查看端口映射信息：docker port CONTAINER

3.8 容器数据拷贝

dockers cp CONTAINER:SRC_PATH DEST_PATH

dockers cp DEST_PATH CONTAINER:SRC_PATH

docker可以避免 crontab 启动脚本时，环境不对的问题，可以在crontab中写

5 * * * * docker start spider
